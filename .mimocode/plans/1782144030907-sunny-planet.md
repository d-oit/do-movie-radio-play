## Task 3: Wire GOAP Orchestrator to Real Pipeline (Issue #96 partial)

**Priority:** Medium — makes GOAP abstraction functional
**Files:** `crates/movie-radio-goap/src/orchestrator.rs`, `crates/movie-radio-goap/src/actions.rs`, `crates/movie-radio-goap/Cargo.toml`

**Approach:**
The current `Action` trait only has `apply(&WorldState) -> WorldState` (state mutation). Need to add execution capability.

1. Extend `Action` trait with `execute()` method:
   ```rust
   #[async_trait]
   pub trait Action: std::fmt::Debug + Send + Sync {
       // ... existing methods ...
       async fn execute(&self, ctx: &mut PipelineContext) -> Result<()>;
   }
   ```
2. Create `PipelineContext` struct holding shared state (paths, config, intermediate results):
   ```rust
   pub struct PipelineContext {
       pub movie_path: PathBuf,
       pub timeline: Option<Timeline>,
       pub gap_analysis: Option<GapAnalysis>,
       pub scripts: Option<Vec<NarrationScript>>,
       pub narration_audio: Option<Vec<AudioSegment>>,
       pub original_audio: Option<Vec<f32>>,
       pub output_path: PathBuf,
       pub sample_rate: u32,
   }
   ```
3. Implement `execute()` for each action struct:
   - `DecodeMovie::execute()` → calls `decode_audio()`
   - `ExtractTimeline::execute()` → calls `extract_timeline()`
   - `IdentifyVisualGaps::execute()` → calls `GapIdentifier::identify_gaps()`
   - `GenerateNarration::execute()` → calls `NarrationGenerator::generate()`
   - `SynthesizeNarrator::execute()` → calls TTS provider
   - `AssembleRadioPlay::execute()` → calls `RadioPlayAssembler::assemble()`
4. Update `orchestrator.rs` to call `execute()` instead of just `apply()`

**Verification:** `cargo test --workspace` + integration test with a small audio file
