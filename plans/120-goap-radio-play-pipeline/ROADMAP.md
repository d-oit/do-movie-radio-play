# Radio Play Generation Pipeline — Roadmap

**Date:** 2026-06-21
**Status:** Proposed (experimental)
**Goal:** Convert any movie into a complete radio play with AI-narrated scene descriptions

## Vision

Input: Movie file (MP4/MKV/WebM)
Output: Complete radio play audio (FLAC/MP3) that is:
- **100% of the original movie audio preserved** (dialogue, SFX, music, ambience)
- **AI narrator voice added** only where a listener would be lost without the picture
- Smooth insertion of narration into natural pauses (never cuts original audio)
- Quality that improves with every run (learns which moments need narration)

**This is automated Audiodeskription** — the narrator describes only what cannot be understood from audio alone.

## Milestone Plan

### Milestone D: GOAP Core + Visual Gap Identification (2 weeks)

**Goal:** Identify WHERE narration is needed — the core intelligence.

| Task | Duration | Output |
|------|----------|--------|
| GOAP planner with A* search | 3 days | `src/goap/planner.rs` |
| World state + action registry | 2 days | `src/goap/mod.rs`, `src/goap/actions.rs` |
| Visual gap identifier algorithm | 5 days | `src/goap/gaps.rs` |
| Gap context analyzer | 2 days | What came before/after, scene boundary detection |
| CLI `radio-play --analyze-only` | 1 day | Shows identified gaps without synthesis |

**Visual Gap Detection Signals (from existing pipeline):**
- Non-voice segments from VAD (candidates for narration)
- Segment tags: pure silence > 3s in dialogue context → likely visual scene
- Audio fingerprint discontinuity → scene change needing orientation
- Long ambience between dialogue blocks → "what's happening" gap
- Existing prompt data (already generated by `src/pipeline/prompts.rs`)

**Acceptance:**
- `timeline radio-play <MOVIE> --analyze-only` outputs gap list with timestamps + confidence
- Not all non-voice segments are marked (only visually-critical ones)
- False-positive rate tracked for learning

### Milestone E: Narration Text Generation (1 week)

**Goal:** Generate brief German descriptions for identified gaps.

| Task | Duration | Output |
|------|----------|--------|
| Narration text generator | 3 days | `src/goap/narrate.rs` |
| Context-aware German descriptions | 2 days | Uses surrounding audio context |
| Timing constraint: text must fit gap | 1 day | Word count bounded by gap duration |
| Template fallback (no LLM needed) | 1 day | Deterministic mode from tags |

**Narration rules:**
- Brief: max ~3 seconds of speech per gap (adjustable)
- German: all output in de-DE
- Contextual: describes only what's visually relevant
- Example: "Er betritt den Raum" / "Stille. Sie sieht ihn an." / "Schnitt: Außenaufnahme, Nacht"

**Acceptance:**
- Every identified gap gets a narration script
- Text length matches available gap duration (at typical speech rate ~150 wpm DE)
- Template mode works without LLM/API

### Milestone F: Voice Synthesis (2 weeks)

**Goal:** Synthesize German narrator voice for each gap.

| Task | Duration | Output |
|------|----------|--------|
| VoiceSynthesizer trait | 1 day | `src/voice/mod.rs` |
| Qwen3-TTS provider (primary DE) | 4 days | `src/voice/qwen3.rs` |
| Orpheus-3B German FT provider | 3 days | `src/voice/orpheus.rs` |
| Kokoro provider (fallback) | 2 days | `src/voice/kokoro.rs` |
| ElevenLabs/OpenAI API providers | 1 day | `src/voice/api.rs` |
| Fallback chain + model download CLI | 1 day | `src/voice/chain.rs` |

**Acceptance:**
- Single consistent narrator voice across entire movie
- German pronunciation and prosody correct
- Emotion matches scene context (tense narration for action, calm for transitions)
- Narration audio duration fits within identified gap

### Milestone G: Radio Play Assembly (1 week)

**Goal:** Insert narrator audio into original movie audio — produce final radio play.

| Task | Duration | Output |
|------|----------|--------|
| Assembly engine | 2 days | `src/goap/assemble.rs` |
| Insert narration into gaps (no cutting) | 2 days | Preserve 100% original audio |
| Crossfade narrator in/out (50-100ms) | 1 day | Smooth transitions |
| Optional time-stretch for tight gaps | 1 day | Expand gap slightly if narration too long |
| Output encoding (FLAC/MP3/WAV) | 1 day | Via ffmpeg or native |

**Key constraint:** Original audio is NEVER cut or removed. Narration is:
1. Inserted in natural pauses (preferred)
2. Overlaid with volume ducking on background (if gap has ambience/music)
3. Gap slightly time-stretched if narration doesn't fit (last resort)

**Acceptance:**
- Output plays without artifacts
- Zero overlap between narrator and original dialogue
- All original content audible and intact
- Duration within 10% of original (some expansion from inserted narration)

### Milestone H: Learning Loop (1 week)

**Goal:** System autonomously gets better at gap identification after every run. No human needed.

| Task | Duration | Output |
|------|----------|--------|
| Execution trace recording (do-memory-core) | 1 day | Episodes in learning DB |
| Automated self-evaluation metrics | 2 days | Overlap, timing, density scoring |
| Autonomous gap identification improvement | 2 days | Pattern-based threshold adjustment |
| Optional human feedback intake (review UI) | 1 day | High-weight correction signal |
| Cross-run pattern accumulation | 1 day | Similar scenes share decisions |

**Self-Learning Signals (fully autonomous):**
- Timing overlap detected → automatically widen gap margins next run
- Narration doesn't fit gap → reduce text length for similar gaps
- Silence gap was narrated but had no preceding context change → deprioritize pattern
- Scene boundary detection improved by comparing fingerprint discontinuities across runs
- Emotion mismatch (calm narration over intense audio) → adjust emotion mapping

**Human Feedback (optional, high-weight when given):**
- User marks narrator segment as "unnecessary" → strong signal to skip similar patterns
- User marks a moment as "missing narration" → strong signal to detect that gap type
- System NEVER waits for human input — continues improving autonomously

**Acceptance:**
- Run N+1 produces measurably better results than Run N (without human input)
- Self-evaluation metrics correlate with actual quality
- Optional human feedback accelerates learning when provided
- `timeline learning-stats --radio-play` shows autonomous improvement curve

## Resource Requirements

### Local-Only Setup (Free)
- Disk: ~4GB for all models (Kokoro + Qwen3 + Orpheus)
- RAM: 4GB minimum (Kokoro), 8GB recommended (Qwen3), 16GB (Orpheus)
- CPU: Any x86-64 or ARM64 (AVX2 recommended)
- GPU: Optional, significantly speeds up Orpheus/Qwen3

### API Setup (Paid)
- ElevenLabs: $5-22/month depending on usage
- OpenAI TTS: ~$15 per hour of generated audio
- No local model storage needed

### Hybrid (Recommended)
- Kokoro for draft/preview (free, fast, CPU)
- ElevenLabs for final render (best quality)
- Budget cap prevents surprise costs

## Self-Learning Philosophy

The system is **fully autonomous by default**:
- Learns from its own execution traces, quality metrics, and pattern recognition
- Evaluates its own output (timing overlaps, silence detection, narration fit)
- Adjusts gap identification thresholds, narration density, and emotion mapping automatically
- **Human feedback is optional** — only used when explicitly provided (review UI corrections)
- The system never blocks on human input; it runs, evaluates, adapts, repeats

Human feedback (when given) is a **high-weight signal** that accelerates learning:
- "This gap didn't need narration" → immediately deprioritize that pattern
- "Narration is missing here" → immediately learn that gap pattern
- But the system continues improving without any human interaction

## Language: German (de-DE) Primary

All narration output defaults to German. The scene description generator and TTS providers must:
- Generate description text in German
- Synthesize speech with German pronunciation and prosody
- Use German emotion descriptions for Qwen3-TTS voice control
- Support the German Orpheus fine-tune (`Orpheus-3b-German-FT-Q8_0.gguf`)
- Config override via `"language": "en"` for other locales

## Open Questions

1. **Gap detection without video**: Audio-only detection will miss some visual moments — acceptable for v1?
   - Proposal: Audio-only v1 with self-learning; optional video frame analysis (VLM) in v2
2. **Narration density**: How much narration is too much?
   - Proposal: Conservative default (only high-confidence gaps), self-adjusting via quality metrics
3. **Time-stretching**: Expanding gaps changes movie timing — acceptable?
   - Proposal: Max 500ms expansion per gap, prefer fitting narration to existing gap length
4. **Subtitle integration**: Use existing SRT for dialogue timing?
   - Proposal: Yes — existing `src/validation/srt.rs` parses subtitles, use as additional gap identification signal

## Experimental Status

This entire pipeline is **experimental** (2026). Expected evolution:
- TTS models improve rapidly — provider trait allows hot-swapping
- GOAP planning may be overkill if failure rates are low — can simplify to linear
- Learning system needs real-world data to validate improvement claims
- First viable end-to-end demo expected within 6 weeks of starting Milestone D
