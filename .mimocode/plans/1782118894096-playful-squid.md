# Plan: Workspace Crate Restructure + Compilation Fixes

## Overview

Restructure the monolithic `movie-nonvoice-timeline` crate into a fine-grained Cargo workspace following the [d-oit/rust-2026-template](https://github.com/d-oit/rust-2026-template) pattern. Fix compilation errors in `kokoro.rs`. Update AGENTS.md, plans/, and GitHub issues.

**Reference**: `d-oit/rust-2026-template` — workspace with `crates/*`, `examples/*`, `benchmarks`, edition 2024, resolver 3, workspace-level `[lints]`, `#![forbid(unsafe_code)]`.

## Phase 3: File Migration

For each crate:
1. Create `crates/<name>/Cargo.toml` with workspace inheritance
2. Create `crates/<name>/src/lib.rs` with `pub mod` declarations
3. Move source files from `src/<module>/` to `crates/<name>/src/`
4. Update `use crate::` paths to `use movie_radio_types::` etc.
5. Change `pub(crate)` to `pub` where needed for cross-crate access
6. Add `#![forbid(unsafe_code)]` to each crate root

### Key visibility changes

- `src/types/` → `movie-radio-types`: All types become `pub`
- `src/error.rs` → `movie-radio-types`: `TimelineError` becomes `pub`
- `src/pipeline/` → `movie-radio-pipeline`: Core functions become `pub`
- `src/voice/` → `movie-radio-voice`: Traits and providers become `pub`
- `src/goap/` → `movie-radio-goap`: Actions, planner, orchestrator become `pub`
- `src/learning/` → `movie-radio-learning`: Database, thresholds become `pub`
- `src/verification/` → `movie-radio-verification`: Verification functions become `pub`
- `src/io/` → `movie-radio-io`: I/O functions become `pub`
- `src/validation/` → `movie-radio-validation`: Validation functions become `pub`

## Risk Mitigation

| Risk | Mitigation |
|------|------------|
| Import path breakage | Update all `use crate::` to cross-crate imports systematically |
| Circular dependencies | Strict dependency graph enforced by cargo-deny |
| `pub(crate)` visibility | Change to `pub` where cross-crate access needed |
| ort API changes | Pin ort version; use documented API only |
| CI breakage | Test CI workflow locally before push |
| Integration tests | Update test imports to use new crate names |
